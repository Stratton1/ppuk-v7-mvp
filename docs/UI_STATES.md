# UI States — Access Components (Frontend)\n\nThis guide covers the presentational components for denied/unavailable states. They do not modify auth, routing, Supabase, or guards—they only render consistent UI.\n\n## Components\n- `components/ui/AccessDenied.tsx`\n  - Use when a user is authenticated but lacks permission (e.g., role/ACL forbids the action).\n  - Props: `title?`, `description?`, optional `actionLabel/actionHref`, optional `dataTestId` for tests.\n  - Pure UI; no redirects or side effects.\n\n- `components/ui/AccessUnavailable.tsx`\n  - Use for neutral/empty/unavailable states (no selection, restricted resource, or missing data).\n  - Props: `title`, `description`, optional `actionLabel/actionHref`, optional `dataTestId`.\n  - Pure UI; no redirects or side effects.\n\n## When to Use\n- AccessDenied: authenticated but forbidden scenarios (403-style messaging), e.g., a protected tab or admin-only section when the user lacks the required role.\n- AccessUnavailable: empty selections, unavailable data, or restricted items where the system cannot show content (e.g., no properties selected, resource not accessible, empty watchlist).\n\n## Testing\n- Provide `dataTestId` when you need deterministic Playwright selectors for these states.\n- Components intentionally avoid reading auth/session state; they only render the supplied copy and links.\n*** End Patch
